Interactive Mode: Type code line by line directly in the Python shell (IDLE). Great for quick experiments.
Script Mode: Write code in a .py file and run the whole file. Used for real programs.

Part 2

print() — Your First Output

⏱ ~5 minutes

The print() function displays output to the screen. It is the most fundamental tool for seeing what your program is doing.

basic_print.py
# Printing a simple message
print("Hello, World!")

# Printing multiple values separated by a comma
print("My name is", "Alice")

# Using sep= to change the separator between values
print("Apple", "Banana", "Cherry", sep=" | ")

# Using end= to change what is printed at the end of the line
print("Hello", end=" ")
print("World")
Output
Hello, World!
My name is Alice
Apple | Banana | Cherry
Hello World
Part 3

Variables & Data Types

⏱ ~10 minutes

A variable is a named storage location in memory. You use it to hold a value that your program can use and change later.

Naming Rules

  • Can contain letters, digits, and underscores (_)
  • Cannot start with a digit — e.g., 2name is invalid
  • Cannot contain spaces — use underscores instead: my_name
  • Case-sensitive: age and Age are two different variables

Common Data Types

TypeKeywordExample

Part 4

User Input with input()

⏱ ~8 minutes

So far our programs used fixed values. The input() function lets the user type a value while the program is running, making programs interactive.

⚠️

Important: input() always returns a string (text), even if the user types a number. If you want to do arithmetic with it, you must convert it using int() or float().

user_input.py
# Taking a name as input (stays as string — no conversion needed)
name = input("Enter your name: ")
print("Hello,", name)

# Taking numbers — must convert to int before doing arithmetic
a = int(input("Enter first number: "))
b = int(input("Enter second number: "))

total = a + b
print("Sum =", total)
Output (example run)
Enter your name: Alice
Hello, Alice
Enter first number: 10
Enter second number: 20
Sum = 30

age    = 20
has_id = True

# Both conditions must be true
print(age >= 18 and has_id)

print(not has_id)

Part 8

Conditional Statements

⏱ ~10 minutes

Conditional statements let your program make decisions — run one block of code if a condition is true, and a different block if it is false.

⚠️

Indentation is mandatory in Python. The code inside an if, elif, or else block must be indented by exactly 4 spaces (or one Tab). Python uses indentation to define code blocks — unlike many other languages that use curly braces { }. Missing or wrong indentation will cause an error.

conditional.py
marks = int(input("Enter your marks: "))

if marks >= 80:
    print("Grade: A")
elif marks >= 60:
    print("Grade: B")
elif marks >= 40:
    print("Grade: C")
else:
    print("Grade: F — Failed")
Output (if marks = 75)
Grade: B

Even or Odd (using Modulus)

even_odd.py
number = int(input("Enter a number: "))

if number % 2 == 0:
    print(number, "is Even")
else:
    print(number, "is Odd")
Output (if number = 7)
7 is Odd

Part 10

🔌 Python Meets Hardware

⏱ ~15 minutes

In your last class you saw how a computer is built — CPU, RAM, ROM, motherboard. Let's now connect those physical components to what you just learned in Python.

A Variable IS a Location in RAM

When you write a = 10 in Python, the value 10 is stored in a real location in your computer's RAM. Python gives that location the name a. You can see the actual memory address of any variable using the built-in id() function.

memory_address.py
a = 10
b = 3.14
c = "Hello"

print("Value of a:", a, "| Memory address:", id(a))
print("Value of b:", b, "| Memory address:", id(b))
print("Value of c:", c, "| Memory address:", id(c))
Output (addresses will differ on every machine)
Value of a: 10 | Memory address: 140720038144936
Value of b: 3.14 | Memory address: 2071023498960
Value of c: Hello | Memory address: 2071024123568
💡

Each variable has a unique address — a number that tells Python exactly where in the RAM chip the value is stored. When you reassign a variable (a = 99), the address changes too, because the value is now stored in a different location.

How Much RAM Does Your Name Use?

Every variable you create takes up actual space in your computer's RAM. Python's built-in sys.getsizeof() function tells you exactly how many bytes a variable occupies in memory. Let's make it personal — type your own name and see how much RAM it needs.

name_in_ram.py
import sys

name = input("Enter your name: ")
print("Your name occupies", sys.getsizeof(name), "bytes in RAM")

# Try a longer name and see the difference
long_name = input("Enter a longer name: ")
print("Longer name occupies", sys.getsizeof(long_name), "bytes in RAM")
Output (example)
Enter your name: Ali
Your name occupies 52 bytes in RAM
Enter a longer name: Alexander
Longer name occupies 58 bytes in RAM
🧠

Each extra character you add increases the size. This is exactly the RAM you saw as a physical stick in the last class — your variables are living inside it right now while the program runs. When the program ends, they disappear. That is why RAM is called volatile memory.

You can also compare different data types to see how each one uses a different amount of memory:

type_sizes.py
import sys

a = 10
b = 3.14
c = "Hello"

print("int  :", sys.getsizeof(a), "bytes")
print("float:", sys.getsizeof(b), "bytes")
print("str  :", sys.getsizeof(c), "bytes")
Output
int : 28 bytes
float: 24 bytes
str : 54 bytes

Seeing How Numbers Are Stored in Binary

You learned in class that RAM stores everything as electrical signals — either ON or OFF. These two states represent 1 and 0. Every number in your program is secretly stored as a sequence of 1s and 0s inside RAM. Python's built-in bin() function lets you see this binary form directly.

💡

What is the 0b prefix? When Python shows a binary number, it adds 0b at the beginning as a label — it stands for "0 binary". It is Python's way of saying "what follows is a binary number, not a regular decimal number." You should ignore the 0b and read only the digits after it. So bin(10) gives 0b1010, which means the actual binary value is 1010.

binary_demo.py
number = int(input("Enter any number: "))

print("Decimal  :", number)
print("Binary   :", bin(number))

# Some examples to understand the pattern
print(bin(1))    # 0b1
print(bin(2))    # 0b10
print(bin(4))    # 0b100
print(bin(8))    # 0b1000
print(bin(10))   # 0b1010
Output (if number = 10)
Decimal : 10
Binary : 0b1010
0b1
0b10
0b100
0b1000
0b1010
🧠

Notice how 1, 2, 4, 8 each add one extra binary digit — this is exactly how RAM stores numbers using bits (binary digits). Each physical memory cell in the RAM chip you saw holds exactly one bit — either 0 or 1.

Estimating the Capacity of Your RAM

You saw a RAM stick in the last class. It is measured in gigabytes (GB). Now that we know how many bytes a single integer variable takes, we can calculate roughly how many integer variables would fill your entire RAM.

ram_capacity.py
import sys

# One integer variable takes 28 bytes in Python
int_size = sys.getsizeof(0)
print("One integer takes:", int_size, "bytes")

# How many bytes are in 8 GB of RAM?
ram_bytes = 8 * 1024 * 1024 * 1024
print("8 GB RAM in bytes:", ram_bytes)

# How many integer variables could fit?
capacity = ram_bytes // int_size
print("Approx. integers that fit in 8 GB:", capacity)
Output
One integer takes: 28 bytes
8 GB RAM in bytes: 8589934592
Approx. integers that fit in 8 GB: 306783378
🤯

Over 300 million integer variables could fit in 8 GB of RAM. This is why RAM size matters for large programs — the more data a program needs to hold at once, the more RAM it consumes. Notice we also used floor division (//) here — a partial variable doesn't count, so we round down to a whole number.
